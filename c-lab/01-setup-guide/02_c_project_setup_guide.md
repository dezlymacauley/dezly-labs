# C project setup guide
_______________________________________________________________________________

Create the directory

```bash
mkdir c-project
```
_______________________________________________________________________________

Enter the directory

```bash
cd c-project
```
_______________________________________________________________________________

Create a `mise.toml` file

```bash
touch mise.toml
```
_______________________________________________________________________________

Give mise permission to read the `mise.toml` file

```bash
trust mise.toml
```
_______________________________________________________________________________

Use mise to install the latest version of `clang`

```bash
mise use clang@latest
```

`clang` will be used to compile the C code.
_______________________________________________________________________________

Use mise to install the latest version of `clang-format`

```bash
mise use clang-format@latest
```

`clang-format` will be used to apply formating to the code in C files.
_______________________________________________________________________________

Use mise to install the latest version of `ninja`

```bash
mise use make@latest
```

`ninja` will use `clang` to build the project, 
and ninja will also create a `compile_commands.json` file that will help 
your lsp `clangd` to understand your project and provide suggestions when
importing header files.
_______________________________________________________________________________

Create the project structure

```bash
mkdir -p include
touch include/lib.h

mkdir -p src
touch src/lib.c
touch src/main.c

touch .clang-format
touch .clangd
touch .gitignore
touch build.ninja
```
_______________________________________________________________________________

Add this to the `include/lib.h`  file

```c
#pragma once

void print_project_name(void);
```
_______________________________________________________________________________

Add this to the `src/lib.c` file

```c
#include <stdio.h>

void print_project_name(void) {
  printf("C Project\n");
}
```
_______________________________________________________________________________

Add this to the `src/main.c` file

```c
#include "lib.h"

int main(void) {
  print_project_name();

  return 0;
}
```
_______________________________________________________________________________

Add this to the `.clang-format` file

```yaml
# This tells clang-format that the file contains C code.
Language: C

# This tells clang-format to format the file using Google's official 
# style guide. Google's style guide was created for C++ files
# but it can be used for C files. 
BasedOnStyle: Google
```
_______________________________________________________________________________

Add this to the `.clangd` file

```yaml
CompileFlags:
  Add: [ 
    -std=c23, 
    -Iinclude,
    -Wall, -Wextra, -Wpedantic
  ]
```
_______________________________________________________________________________

Add this to the `.gitignore` file

```gitignore
# Improves the performance of the clangd language server
# Generated by `clangd` when you open files in the project
.cache/ 

# Build output
# Generated by `ninja` when your run `mise build`
build/
.ninja_log

# Improved language support for clangd
# Generated by `ninja` when you run `mise build`
compile_commands.json
```
_______________________________________________________________________________

Add this to the `build.ninja` file

```ninja
# Compiler settings.
cc = clang
cflags = -std=c23 -Iinclude -Wall -Wextra -Wpedantic

# Rules.
rule compile
    command = $cc $cflags -c $in -o $out

rule link
    command = $cc $in -o $out

# Build object files.
build build/obj/lib.o: compile src/lib.c
build build/obj/main.o: compile src/main.c

# Build executable.
build build/bin/main: link build/obj/lib.o build/obj/main.o

# Default target.
default build/bin/main
```
_______________________________________________________________________________

Your `mise.toml` file should already have this

```toml
[tools]
clang = "latest"
clang-format = "latest"
ninja = "latest"
```
_______________________________________________________________________________

Add this to the end of your `mise.toml` file,
so that the full file looks like this

```toml
[tools]
clang = "latest"
clang-format = "latest"
ninja = "latest"

[settings]
task.timings = false

[tasks.build]
description = "🔨 Build the project with ninja" 
quiet = true
run = """
ninja --quiet
ninja -t compdb > compile_commands.json
"""

[tasks.dev]
description = "🇨 Run the project" 
quiet = true
run = """
mise build
./build/bin/main
"""

[tasks.clean]
description = "🧼 Clean the project directory" 
quiet = true
run = """
rm -rf .cache/ build/ .ninja_log compile_commands.json
"""
```
_______________________________________________________________________________

To view the list of `mise tasks`, run this command

```bash
mise tasks
```
_______________________________________________________________________________

You should see this output

```
Name   Description
build  🔨 Build the project with ninja
clean  🧼 Clean the project directory
dev    🇨 Run the project
```
_______________________________________________________________________________

To run a task the syntax is:
`mise name-of-task`
