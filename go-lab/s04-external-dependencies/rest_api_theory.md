REST = Representational state transfer

It is a set of architectural constraints. It is not a protocol or a standard.

When the client makes a request, the REST API transfers a representation
of the state of the resource to the requester or endpoint.

Data is delivered via HTTP. Usually as JSON (JavaScript Object Notation),
or text.

_______________________________________________________________________________

A REST API meets the following criteria.

- Client-Server Architecture where requests are managed via HTTP.

- Stateless Client-Server communication. This means that each request is
separate. Data is not stored on the server.

- A uniform interface.
